      API keys & sources
      
Store your enrichment API keys so SOC Mate can run live lookups against VirusTotal, AbuseIPDB, and ipinfo. Keys are saved locally in Chrome storage and never transmitted except to the respective API. Optionally encrypt them with a passphrase so they aren't visible in plain text via DevTools.

      
Live lookups send the indicator to a third party (VirusTotal / AbuseIPDB / ipinfo / RDAP). Keys stored locally only.

      Enrichment behavior
      
Per-provider parallelism + minimum delay between requests. Defaults are conservative (free-tier safe). The loop also auto-throttles to 1-at-a-time if a server actually returns 429.


      

        
        
      


      
      

        
        
      

      
Free tier: 4/min → 1 parallel + 16s delay (default). Paid: 10-15 parallel + 0s.


      
      

        
        
      

      
Free: 1000/day @ ~1/sec → 1 parallel + 1s delay (default). Premium: 5-10 parallel + 0s.


      
      
      
RDAP and ipwho.is have no published limit but be polite. ipinfo (if token set) is 50k/month.


      
      
      
Single-result Copy always uses the short summary regardless of this setting.

      Auto-extraction sites
      
Extra domains or URLs where the Auto toggle also fires — in addition to the built-in Splunk and Defender detection. One entry per line. Lines starting with # are ignored.

        qradar.corp.local

      Extraction sources
      
Extraction blocklist — domains whose pages SOC Mate will never auto-extract from. Add reference sites like virustotal.com or abuseipdb.com here: their pages are full of IOCs from other cases and would pollute your results. Matches the exact domain and all subdomains. One entry per line.

      
      

        
        
        
      

    


    

      Redaction list
      
Words/phrases to strip from output when the Redact toggle is on (Extract tab). Case-insensitive. One per line. Useful for hiding customer or internal names before sharing copied IOCs.

      
      
      
      
      

        
        
      

    


    

      Default output
      
Set the default state of the Extract tab every time SOC Mate opens — which toggles are on, which separator is active, and whether category headers appear in the All IOCs pane. These are just starting values; you can always change them on-the-fly during an investigation without affecting these saved defaults.

      

        
        
        
        
        
        
        
        
        
        
      

      

        
        
      

    


    

      Splunk security controls
      
Define your security controls (proxy, firewall, Sysmon, etc.). For each one set the index, sourcetype, and the Splunk field name per IOC type. Leave a field blank to exclude that IOC type from this control. When you click OR / AND / IN on an IOC card, one sub-query is generated per matching control — giving you a ready-to-paste multi-source hunt.

      

      

        
        
        
        
      

    


    

      IOC field names & log locations (simple / single-source)
      
Simple fallback used when no security control above covers an IOC type. Pre-fill the Splunk field name, index, and sourcetype — index/sourcetype are prepended when you copy a query.

      

      

        
        
        
      

    


    

      Defender KQL — tables & columns
      
Define your Defender Advanced Hunting data sources. For each table set the column and operator per IOC type. Leave a field blank to exclude that IOC type from this table. When you click Defender on an IOC card, one sub-query is generated per matching table — giving you a ready-to-paste multi-table hunt.

      
Operators: in~ (case-insensitive exact — best for IPs/hashes), in (case-sensitive exact), has_any (token match — best for domains/URLs/paths).

      

      

        
        
        
        
      

    


    

      Defender KQL — simple fallback (single table)
      
Simple fallback used when no KQL table above covers an IOC type. Pre-fill the table, column, and operator for each IOC category.

      
Operators that work with multiple values: in~ (case-insensitive exact match — best for IPs/hashes), in (case-sensitive exact), has_any (token match — best for domains/URLs/paths).

      

      

        
        
        
      

    


    

      Categories
      
Control which IOC categories appear in extraction results. Hiding a category skips it entirely — it won't show up in cards, the All IOCs pane, queries, or exports. Useful if your environment doesn't produce certain indicator types (e.g. email subjects, registry keys) and you want a cleaner, faster output.

      

      

        
        
      

    


    

      History
      
SOC Mate keeps a running log of every unique IOC it extracts. The History tab lets you search, copy, and export past indicators — handy for building timelines or recovering IOCs from closed tabs. Here you can cap how many entries to retain, choose whether to record the source page URL alongside each indicator, and switch between newest-first and type-grouped views.

      
      
      
      
      
      

        
        
        
      

    


    

      Report templates
      
Build named ticket templates for different alert types. Each template defines the fields in order. The IOCs (auto) type auto-fills from your current extraction with enrichment verdicts inline.

      

      

        
        
        
        
      

    


    

      Team config — export & import
      
Share your Splunk controls, KQL tables, and field settings with teammates. Export once, share the file, everyone else imports it.
